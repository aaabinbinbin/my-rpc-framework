# RPC 框架实现教程 - 课程大纲

## 课程概述

本课程将带你从零开始实现一个基于 Java 的完整 RPC 框架，涵盖网络通信、序列化、服务注册与发现等核心技术。

---

## 为什么选择这些技术栈？

### 1. **Java** 
- **优势**：跨平台、生态丰富、企业级应用广泛
- **适合 RPC**：强大的网络编程支持、完善的并发处理机制

### 2. **Netty** （核心网络框架）
- **为什么选择 Netty 而不是原生 NIO？**
  - ✅ **简化开发**：封装了复杂的 NIO 细节，API 简洁易用
  - ✅ **高性能**：经过优化的 Reactor 线程模型，性能卓越
  - ✅ **稳定性**：阿里巴巴、腾讯等大厂都在使用，经受过生产环境考验
  - ✅ **功能丰富**：内置多种编解码器、协议支持
  - ✅ **社区活跃**：文档完善，问题容易找到解决方案
- **对比原生 NIO**：原生 NIO API 繁琐，需要处理 Selector、Channel、Buffer 等底层细节，容易出错

### 3. **NIO** （非阻塞 IO）
- **为什么选择 NIO 而不是 BIO？**
  - ✅ **高并发**：单线程可以处理多个连接，适合高并发场景
  - ✅ **资源利用率高**：不需要为每个连接创建独立线程
  - ✅ **扩展性好**：轻松应对成千上万的并发连接
- **BIO vs NIO**：BIO 一个连接一个线程，资源消耗大；NIO 多路复用，更适合 RPC 框架

### 4. **ZooKeeper** （服务注册与发现）
- **为什么选择 ZooKeeper？**
  - ✅ **强一致性**：保证分布式系统数据一致性
  - ✅ **高可用**：集群部署，自动故障转移
  - ✅ **临时节点**：服务下线自动删除注册信息
  - ✅ **监听机制**：实时感知服务变化
  - ✅ **成熟稳定**：Apache 顶级项目，广泛应用于分布式协调
- **替代方案对比**：
  - **Nacos**：功能更强大，支持配置管理，但相对重量级
  - **Eureka**：已停止更新，不推荐新项目使用
  - **Consul**：Go 语言开发，支持多数据中心，但国内使用较少
  - **结论**：ZooKeeper 学习成本低，资料丰富，非常适合学习 RPC 原理

### 5. **其他将要用到的技术**
- **Kryo/Hessian**：高效序列化框架（比 Java 原生序列化性能好 10 倍以上）
- **Spring/Spring Boot**：可选，用于简化使用和集成
- **Maven**：项目构建和依赖管理
- **SLF4J + Logback**：日志框架

├── example-api/                 # 示例接口定义
├── example-provider/            # 服务提供者示例
└── example-consumer/            # 服务消费者示例

## 学习建议

1. **循序渐进**：按照课程顺序学习，不要跳步
2. **动手实践**：每节课都要亲自编写代码
3. **理解原理**：不仅要会用，更要理解为什么这样设计
4. **调试分析**：学会使用 Wireshark、Netty 日志等工具分析问题
5. **持续优化**：完成基础功能后，尝试性能优化
